Security

Protecting your data and assets

At PMT Markets, security is fundamental to our operations. We employ a defense-in-depth strategy with multiple layers of protection to safeguard user data, financial assets, and platform integrity.

1. Data Protection & Encryption

We implement strong encryption protocols to protect data throughout its lifecycle:

  • Transport Layer Security (TLS 1.3): All data transmitted between users and our Platform is encrypted in transit
  • At-Rest Encryption: Sensitive data is encrypted using AES-256 encryption when stored
  • Key Management: Encryption keys are managed using hardware security modules (HSMs) or cloud key management services
  • Database Encryption: Personally identifiable information (PII) and financial data are encrypted at the database level

2. Infrastructure Security

Our infrastructure is designed with security as a primary consideration:

  • Cloud Security: Leveraging secure cloud infrastructure with built-in security controls
  • Network Segmentation: Isolated network segments to limit lateral movement
  • Firewalls & WAF: Web application firewalls and network firewalls to filter malicious traffic
  • DDoS Protection: Mitigation services to protect against distributed denial-of-service attacks
  • Intrusion Detection/Prevention: Systems to detect and block malicious activity

3. Access Control & Authentication

We enforce strict access controls to prevent unauthorized access:

  • Multi-Factor Authentication (MFA): Required for all user accounts and privileged system access
  • Role-Based Access Control (RBAC): Least privilege principle for employee access
  • Privileged Access Management: Just-in-time access for administrative functions
  • Session Management: Secure session handling with automatic timeout
  • Biometric Authentication: Optional biometric verification for mobile access

4. Application Security

Our development process incorporates security at every stage:

  • Secure Development Lifecycle: Security requirements integrated from design through deployment
  • Code Review: Peer reviews with security checklists
  • Static & Dynamic Analysis: Automated security testing of code and running applications
  • Dependency Scanning: Regular scanning for vulnerable third-party libraries
  • API Security: Rate limiting, input validation, and proper authentication for all APIs

5. Financial Security

Special measures to protect financial assets and transactions:

  • Cold Storage: Majority of digital assets stored in offline, air-gapped wallets
  • Multi-Signature Wallets: Requiring multiple approvals for significant transactions
  • Transaction Monitoring: Real-time monitoring for suspicious financial activity
  • Withdrawal Controls: Time delays and additional verification for large withdrawals
  • Insurance Coverage: Cyber insurance and crime insurance policies

6. Monitoring & Incident Response

Proactive monitoring and prepared response capabilities:

  • 24/7 Security Operations Center (SOC): Continuous monitoring of security events
  • Security Information & Event Management (SIEM): Centralized logging and correlation
  • Incident Response Plan: Documented procedures for security incidents
  • Forensic Capabilities: Tools and processes for investigating security events
  • Regular Drills: Tabletop exercises to test incident response readiness

7. Third-Party Security

We ensure our vendors and partners meet our security standards:

  • Vendor Risk Assessment: Security evaluation of third-party providers
  • Contractual Security Requirements: Security clauses in vendor agreements
  • Regular Audits: Security assessments of critical vendors
  • Data Processing Agreements: Compliance with data protection requirements

8. Security Testing & Assurance

Regular testing to validate our security controls:

  • Penetration Testing: Regular external and internal penetration tests
  • Vulnerability Scanning: Automated scanning of networks and applications
  • Red Team Exercises: Simulated attacks to test detection and response
  • Bug Bounty Program: Rewards for security researchers who report vulnerabilities
  • Security Audits: Independent third-party security assessments

9. Compliance & Certifications

We adhere to industry security standards and frameworks:

  • ISO 27001 Information Security Management System
  • SOC 2 Type II compliance for security controls
  • PCI DSS compliance for payment processing
  • GDPR and other data protection regulations

10. Security Contact & Responsible Disclosure

We encourage responsible disclosure of security vulnerabilities:

  • Report security issues to: security@pmtmarkets.com
  • PGP key available for encrypted communications
  • We commit to acknowledging reports within 24 hours
  • We will not take legal action against researchers acting in good faith
  • We offer rewards for qualifying vulnerabilities through our bug bounty program

Note: Security is an ongoing process. We continuously evaluate and improve our security measures to address evolving threats.
